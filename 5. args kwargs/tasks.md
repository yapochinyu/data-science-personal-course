
## Ключевые идеи

1. Один оператор `*` — две противоположные операции: в определении функции — сборка (packing) лишних аргументов в контейнер, в вызове — распаковка (unpacking) контейнера в отдельные аргументы.
2. `*args` собирает произвольное число позиционных аргументов; внутри функции `args` — всегда `tuple`, даже при одном аргументе или нуле (пустой кортеж).
3. `**kwargs` собирает произвольное число именованных аргументов; внутри функции `kwargs` — всегда `dict`. Имена `args`/`kwargs` — соглашение, синтаксис требует только звёздочек.
4. Строгий порядок в сигнатуре: обычные позиционные → параметры с дефолтом → `*args` → keyword-only → `**kwargs`; нарушение — `SyntaxError`.
5. Всё после `*args` или после голой `*` в сигнатуре — keyword-only аргументы: передаются только по имени, позиционная передача даёт `TypeError`. Голая `*` — способ сделать API явным (`sorted(..., key=, reverse=)`).
6. При вызове `f(*lst)` распаковывает список в позиционные аргументы, `f(**d)` — словарь в именованные; позиционная распаковка идёт перед именованной; `zip(*matrix)` — та же механика.
7. Распаковка `**d` требует точного соответствия ключей параметрам: лишний ключ даст `TypeError: unexpected keyword argument`, если функция не имеет `**kwargs`.
8. Extended unpacking в присваивании: `a, *rest, b = seq` — звёздочка собирает «всё остальное» в **список** (не кортеж!); может быть пустым; двух звёздочек в одном присваивании быть не может.
9. Дефолтное значение параметра вычисляется ровно один раз — при выполнении `def` — и хранится в `func.__defaults__`; это ключевой факт всей темы.
10. Отсюда ловушка изменяемого дефолта: `def f(x, lst=[])` — один общий список на все вызовы, мутации накапливаются; с неизменяемыми дефолтами (`int`, `str`, `None`) проблемы нет, потому что «изменение» создаёт новый объект.
11. Правильный паттерн: дефолт `None` + `if lst is None: lst = []` внутри тела; вариант `lst = lst or []` — с подвохом: он подменит и явно переданный пустой список.
12. Связка `def wrapper(*args, **kwargs): return func(*args, **kwargs)` — универсальный проброс: так декораторы и обёртки работают с любой сигнатурой; в ML так пробрасывают гиперпараметры (`Model(**params)`, конфиг из JSON).

### Задача 14 (идеи 9, 10) — рассуждение вслух (вопрос-маркер)

Интервьюер: «Почему в Python вообще возможен баг с изменяемым дефолтом? Это же очевидный промах дизайна — почему дефолты не вычисляются при каждом вызове?» Дай ответ уровня уверенного кандидата: когда именно вычисляется дефолт, где он хранится, почему для `n=0` и `s=''` проблемы нет, и одно следствие того, что `def` — исполняемая инструкция.

> [!hint]- Подсказка 1 
> Опорные факты: `def` — не декларация, а исполняемый код; дефолты — часть создания объекта-функции; хранилище — `__defaults__`.

> [!hint]- Подсказка 2 
> Для неизменяемых дефолтов «изменение» внутри функции — это перепривязка локального имени на новый объект, дефолт в `__defaults__` не трогается. Мутация же меняет сам хранимый объект.

> [!hint]- Подсказка 3 
> Структура ответа: (1) дефолт вычисляется один раз при выполнении `def`, потому что def создаёт объект-функцию со всеми атрибутами; (2) лежит в `func.__defaults__`; (3) неизменяемые безопасны — перепривязка vs мутация; (4) следствие исполняемости def: дефолт может быть вызовом функции — `def f(t=time.time())` зафиксирует время определения, не вызова.
