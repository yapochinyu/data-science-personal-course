## Ключевые идеи

1. Переменная в Python не хранит значение — она хранит ссылку на объект в куче; присваивание `b = a` не создаёт новый объект, а привязывает второе имя к тому же объекту и увеличивает счётчик ссылок.
2. `==` сравнивает значения (вызывает `__eq__`), `is` сравнивает идентичность (`id()`, один ли это физический объект); два разных объекта могут быть равны по `==`, но не по `is`.
3. `None` проверяют через `is`, потому что `None` — синглтон: сравнение указателей быстрее и не может быть переопределено чужим `__eq__`.
4. Операция над неизменяемым объектом (`int`, `str`, `tuple`…) всегда создаёт новый объект и перепривязывает имя — поэтому «разделяемая ссылка» для них безопасна; мутация изменяемого объекта (`list`, `dict`, `set`, ndarray, DataFrame) видна через все имена, ссылающиеся на него.
5. Водораздел «мутация vs новый объект»: `sort()/append()/extend()` меняют объект на месте и возвращают `None`; `sorted()/lst + [x]/срез` создают новый объект. `x = lst.sort()` кладёт в `x` `None` — это намеренный дизайн.
6. Shallow copy (`lst.copy()`, `lst[:]`, `dict(d)`, `copy.copy`) копирует только внешний контейнер; внутренние объекты остаются общими ссылками — мутация вложенного элемента затрагивает оригинал.
7. Deep copy (`copy.deepcopy`) рекурсивно копирует все вложенные изменяемые объекты; стоит O(n) по времени и памяти; циклические ссылки обрабатываются через словарь `memo` (`id(original) → copy`).
8. `[[0]*3]*3` — ловушка: внешний `*` копирует ссылки, все «строки» — один список; правильно — `[[0]*3 for _ in range(3)]`. При этом `[0]*3` безопасен, потому что `0` неизменяем.
9. Изменяемый аргумент по умолчанию (`def f(x, acc=[])`) создаётся один раз при `def` и живёт в `__defaults__`; мутации накапливаются между вызовами. Правильный паттерн — `acc=None` + создание внутри тела.
10. Функция, мутирующая входной аргумент, создаёт побочный эффект: повторный вызов на тех же данных даёт другой результат; профессиональная норма — не мутировать входы либо явно это документировать.
11. NumPy-срез (`a[1:4]`, `a[::2]`) возвращает view — общую память с оригиналом; запись в view меняет исходный массив. Fancy indexing и булева маска возвращают копию. Срез обычного list всегда даёт новый список.
12. В pandas `df2 = df` — вторая ссылка; фильтрация может вернуть view или copy (отсюда `SettingWithCopyWarning`); для независимости — явный `.copy()`.
13. CPython кэширует int от -5 до 256, поэтому `a is b` для малых чисел даёт `True` — но это деталь реализации; числа сравнивают только через `==`.
14. Память освобождается через reference counting (счётчик = 0 → объект удаляется сразу) плюс циклический GC; `del a` удаляет имя, а не объект.
