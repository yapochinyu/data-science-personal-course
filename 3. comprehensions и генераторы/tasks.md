## Ключевые идеи

1. Comprehension — синтаксический сахар для построения коллекции из итерируемого в одно выражение; быстрее цикла с `append` на 20–30% за счёт байт-кода (нет поиска и вызова метода `append` на каждой итерации).
2. Видов comprehension три: list `[...]`, set `{...}`, dict `{k: v ...}`; круглые скобки `(x for x in ...)` дают не кортеж, а генераторное выражение — tuple comprehension не существует.
3. Две позиции `if`: после `for` — фильтр (отсеивает элементы, `else` невозможен); перед `for` — тернарный оператор (преобразует каждый элемент, `else` обязателен). Фильтр меняет количество, тернарник — значения.
4. Вложенные `for` в comprehension читаются слева направо как вложенные циклы: первый `for` — внешний, второй — внутренний.
5. Список — энергичное вычисление: вся коллекция материализуется в памяти сразу, O(n) памяти. Генератор — ленивое: элемент вычисляется только по запросу, O(1) памяти.
6. Генератор хранит не элементы, а состояние выполнения: ссылку на источник, локальные переменные и точку остановки; `next()` восстанавливает состояние, доходит до `yield`, отдаёт значение и замораживается.
7. Функция с `yield` — генератор-функция: при вызове код не выполняется, возвращается объект-генератор. `return` завершает функцию один раз, `yield` приостанавливает с сохранением состояния и позволяет отдавать значения многократно.
8. `yield from` делегирует вложенному генератору — идиоматичный способ рекурсивных генераторов (например, flatten вложенных списков).
9. Генератор одноразовый: после исчерпания повторный проход тихо даёт пустоту (не ошибку) — источник тихих багов вида `sum(gen)` затем `max(gen)`.
10. `next(gen)` вручную запрашивает элемент; при исчерпании летит `StopIteration`; `for` ловит его сам; `next(gen, default)` возвращает default вместо исключения.
11. Генератор не поддерживает `len()`, индексацию и срезы — потому что не хранит элементы; ленивость покупается ценой функциональности.
12. Экономия памяти ≠ экономия времени: при полном проходе генератор не быстрее списка (иногда чуть медленнее); его выигрыш — память и возможность раннего выхода (`next((x for x in data if cond), None)` останавливается на первом совпадении).
13. Выбор: список — многократный проход, индекс, `len`, срезы, малые данные; генератор — один проход, большие/бесконечные/потоковые данные, пайплайны, ранний выход.
14. В ML генераторы — основа потоковой обработки: чтение гигабайтных файлов построчно (файловый объект — сам генератор строк), батчинг, цепочки ленивых трансформаций; так концептуально устроен DataLoader.
15. Переменная comprehension в Python 3 не «протекает» наружу — живёт в своей области видимости (в Python 2 протекала).
