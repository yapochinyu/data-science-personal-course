## Ключевые идеи

1. `lambda аргументы: выражение` — анонимная функция из одного выражения без явного `return`; внутри нельзя присваивания, циклы, try/except; годится только для крошечных одноразовых функций, передаваемых на месте.
2. Функции в Python — объекты первого класса: их можно присваивать переменным, класть в коллекции, передавать аргументами и возвращать из функций — именно поэтому существуют `key=`, `map`, `filter`.
3. Передаётся функция, а не её вызов: `key=len` — ссылка на объект-функцию, которую `sorted` вызовет сам для каждого элемента; `key=len()` — ошибка, потому что это вызов здесь и сейчас.
4. `map(func, iterable)` применяет функцию к каждому элементу; умеет несколько iterable сразу (`map(f, a, b)` — f получает пары).
5. `filter(func, iterable)` оставляет элементы, где func вернула truthy; `filter(None, data)` выбрасывает все falsy-значения (`0`, `''`, `None`, `[]`…).
6. В Python 3 `map`/`filter`/`zip`/`enumerate` возвращают ленивые одноразовые итераторы, а не списки — без `list()` увидишь `<map object>`, а после прохода итератор пуст.
7. `map`/`filter` с lambda почти всегда проигрывают comprehension по читаемости; `map(int, strings)` с готовой функцией — легитимный краткий вариант.
8. `zip(a, b, ...)` склеивает последовательности поэлементно в кортежи и молча обрезает по самой короткой — потеря данных без ошибки; для заполнения хвоста — `itertools.zip_longest(fillvalue=...)`.
9. `zip(*matrix)` транспонирует матрицу: `*` распаковывает строки как отдельные аргументы, `zip` собирает i-е элементы каждой строки в кортеж-столбец.
10. `enumerate(iterable, start=0)` даёт пары `(индекс, элемент)` — идиоматичная замена антипаттерну `for i in range(len(data))` (двойное обращение, красный флаг на собесе).
11. `sorted(iterable)` возвращает новый список, не трогая исходный; `.sort()` мутирует список на месте и возвращает `None` — `y = x.sort()` кладёт в `y` `None`.
12. `key=` — функция «по какому признаку сравнивать»: `key=len`, `key=str.lower`, `key=lambda p: p['age']`; сравниваются результаты key, а не сами элементы.
13. Многоуровневая сортировка — key возвращает кортеж: `key=lambda x: (x['age'], x['name'])`; кортежи сравниваются поэлементно. Для убывания числового поля — минус: `(-x['score'], x['name'])`; для строк минус не работает.
14. Сортировка Python стабильная: элементы с равными ключами сохраняют исходный порядок — это позволяет сортировать в несколько проходов (сначала вторичный ключ, потом первичный) и решает случай «строка по убыванию».
